## 3. 개발자에게 전달하면 좋을 발견사항

1. **백엔드(Supabase) 미설정 시 페이지가 멈춤** — 환경변수가 없을 때 쓰라고 만든
   임시(stub) 코드가 `.select().eq()`처럼 연결된 쿼리에서 깨집니다. 그래서 Supabase
   없이는 디자인 미리보기조차 어렵습니다. (`app/lib/supabase.js`의 stub 보완 권장)
2. **빈 데이터 방어 부족** — 예: `Professor` 페이지는 교수 레코드가 없으면
   `Cannot read properties of null (reading 'experience')`로 500 발생. 데이터 입력
   전 단계에서 빈 값 처리(옵셔널 체이닝 등) 추가 권장.
3. **린트 32건** — `react-hooks` 규칙, 설정파일의 `process` 미정의 등. 빌드는 되지만
   정리해두면 좋음.

## 4. PC에서 실행하는 법

### 사전 준비 (최초 1회)
- **Node.js 22** 설치 (저장소 `.nvmrc` 기준)
- 터미널에서 pnpm 활성화: `corepack enable`

### A. 디자인·동작까지 제대로 보려면 (권장)
README의 1~5단계대로 **Supabase 백엔드**를 먼저 연결해야 합니다. 요약:
1. https://supabase.com 가입 → New Project (Region: Seoul)
2. SQL Editor에서 `supabase/migrations/` 의 SQL을 **번호 순서대로** 실행
3. 프로젝트 루트에 `.env.local` 파일 생성 후 아래 입력:
   ```env
   VITE_SUPABASE_URL=https://<프로젝트ID>.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJ...   # Supabase Settings → API 에서 복사
   ```
4. 터미널에서:
   ```bash
   cd C:\LabProject\phi-lab-homepage
   pnpm install
   pnpm dev          # 브라우저에서 http://localhost:5173 접속
   ```
5. (선택) 초기 콘텐츠 일괄 입력: `.env.local`에 service_role 키까지 넣고
   `node scripts/seed-content.mjs` 실행 → 교수/멤버/논문 등이 채워집니다.

### B. 명령어 요약
```bash
pnpm dev        # 개발 서버 (http://localhost:5173)
pnpm build      # 배포용 빌드 (build/client 에 출력)
pnpm preview    # 빌드 결과 미리보기
```

> 관리자 로그인(Google)·배포(Render)까지의 전체 과정은 저장소의 `README.md` 3~6단계 참고.
