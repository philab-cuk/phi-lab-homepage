# PHI Lab homepage — 프로젝트 규칙

## DB 변경 규칙 (반드시 지킬 것)

로컬 Supabase 스택을 직접 손대다 마이그레이션 파일을 빠뜨려 운영 반영이 누락된 사고가 있었음. 재발 방지를 위해 아래를 강제한다.

1. **직접 ALTER 금지 — 스키마 변경은 마이그레이션 파일로 먼저.**
   `docker exec ... psql -c "ALTER TABLE ..."` 같은 즉석 스키마 변경을 하지 않는다.
   `supabase/migrations/` 에 SQL 파일을 먼저 만들고(`alter table ... add column if not exists ...` 형태), 그 SQL 을 로컬 DB에 1회만 적용한다. 파일과 DB가 항상 같이 간다.

2. **커밋 전 스키마 드리프트 0 확인.**
   `supabase db diff` 를 돌려 출력이 "No schema changes found" 인지 확인한다. "파일엔 없는데 DB에만 있는 변경"이 남아 있으면 커밋하지 않는다.

3. **데이터(콘텐츠) 변경은 멱등 시드 스크립트로.**
   역할·이름 등 콘텐츠성 데이터 변경은 즉석 SQL 대신 `scripts/seed-*.mjs` 멱등 스크립트로 작성한다(여러 번 실행해도 동일). 운영에도 그 스크립트로 반영한다.

4. **`supabase db reset` 사용 금지.**
   로컬 DB에는 마이그레이션에 없는 데이터(시드·콘텐츠)가 직접 들어 있어, reset 하면 전부 사라진다. 절대 쓰지 않는다.

## 언어 규칙 (콘텐츠·UI 기본 언어)

사이트 텍스트의 기본 언어 방향. 다음 세션들도 콘텐츠·UI 를 만지거나 새로 만들 때 이 기준을 따른다.

- **News · Posts → 한글이 기본(default).** 소식/게시판 본문·제목은 한국어를 우선한다. (영문 혼용 가능)
- **공개 사이트(방문자용) → 영어가 기본(default).** 메뉴·버튼·섹션 제목·역할 라벨·멤버/연구/강의/논문 등 방문자에게 보이는 UI·콘텐츠는 영어를 우선한다. (한글 혼용 가능 — 예: 이름은 "김효정 HyoJung Kim")
- **관리자 화면(admin, `/admin` · `app/pages/admin/*`, 공통 `AdminUI`) → 한글이 기본(default).** 운영자만 보는 내부 도구라 한국어를 우선한다. 버튼·헤더·힌트·뱃지·모드 표시(예: "삭제 모드", "보기·읽기 전용", "편집 중") 등 admin UI 문구는 한글로 쓴다. (영문 혼용 가능)
  - 단, admin 에서 **편집해 공개 페이지에 그대로 나가는 값**(멤버 역할 라벨 `Principal Investigator`·`Undergraduate Members`·`Research Engineer`, 멤버 이름, 연구/논문 제목 등)은 위 공개 사이트 규칙(영어 기본)을 따른다. admin 도구 문구(한글)와 구분할 것.
- "혼용 가능"은 필요할 때 두 언어를 함께 쓸 수 있다는 뜻이지, 기본 언어를 바꿔도 된다는 뜻이 아니다. 새 라벨/문구를 추가할 때는 그 영역의 기본 언어를 먼저 따른다.
  - 예: 대학원 과정명 등 한국 학제 고유 표현은 한글 유지 가능.

## 관리자 초대·연결 흐름 (정본)

멤버와 로그인 계정을 잇는 방식. 다음 세션이 admin 초대/계정 관련 코드를 만질 때 이 모델을 따른다.

- **연결 고리는 이메일 하나.** `members.email == 로그인 이메일` 이면 연결된 것. `redeem_invite` 는 `admin_users`(화이트리스트)만 추가하고 `members` 는 건드리지 않는다 — 연결은 이메일 일치로 성립.
- **역할이 두 종류 (혼동 주의).** `admin_users.role` = 권한(admin/professor=편집자, researcher/alumni=본인만) vs `members.role` = 화면 표시(Undergraduate Members 등). 서로 다른 개념.
- **모든 초대의 정본 = Members 화면.** 멤버 생성(최소: 이름+이메일) → "초대 링크 생성"(항상 `researcher`) → 그 이메일로 로그인 → 화이트리스트 등록 + 이메일 일치로 자동 연결. 별도 "연결 계정" 수동 지정 없음.
- **Users/Invites = 계정·권한 관리 + 초대 현황 (초대 발급 UI 없음).** 가입한 계정의 권한을 admin/professor 로 올리는 건 여기서. 멤버 없이 임의 이메일을 직접 초대하는 옛 경로는 제거됨(멤버 초대로 일원화).
- **멤버 연결 상태**: 이메일 없음 / 초대 전(이메일 있고 초대·가입 전) / 초대 대기(활성 초대 있음) / 연결됨(admin_users 에 그 이메일 존재).
- **캐비어트**: 일원화 결과, 멤버 카드 없는 순수 관리자도 멤버로 만들어야 하고 그 멤버는 공개 페이지에 노출된다. 비공개/스태프 계정이 필요해지면 members 에 hidden 상태를 새로 도입할 것.

## 검증/테스트
- UI 변경은 `pnpm build` 통과 + Playwright(MCP)로 실제 화면 확인.
- 검증할 때 실제 운영/실데이터 레코드를 테스트용으로 수정하지 않는다. 임시 더미를 만들어 쓰고 끝나면 지운다.
